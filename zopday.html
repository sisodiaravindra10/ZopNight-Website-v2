what ships in beta

The deployment layer without the YAML cliff.

Most "deployment platforms" are config-file factories. ZopDay is a state machine you can read in a single screen.

live in beta

Pipeline runner

Triggers on push, tag, or schedule. Runs against AWS, GCP, Azure, or your own kubeconfig. No agent install — pulls from your registry over a signed token.

live in beta

Dependency-aware deploys

Database migrations gate API rollout. API rollout gates frontend. Failed migration? The whole flight is held. No more "front-end deployed, back-end didn't" Slack threads.

Q2 2026

Release management

Canary, blue-green, progressive rollout out of the box. Health checks plug into the same metrics ZopNight uses for right-sizing — one source of truth for "is this thing OK?"

Q3 2026

Auto-rollback

SLO breach → automatic rollback to the last known-good. The threshold is yours; the response is instant. Pages your on-call after the fix has shipped.

why we're building it

Because the "deploy with confidence" tools don't.

01

Two systems, one source of truth.

ZopNight already knows your inventory, dependencies, and metrics. ZopDay deploys onto that map. No "import your services" tax — we already have them.

02

Multi-cloud from line one.

If your stack spans AWS + GCP, your deploy tool shouldn't make you pick one. ZopDay treats each provider as a target, not a religion.

03

The pipeline reads like English.

Your release plan is a single declarative file, ~30 lines, no Helm charts to maintain. Reviewers can understand it. Your CFO can understand it.

04

Cost-aware deploys.

Deploying a heavier instance? ZopDay flags it before the apply runs. Deploying overnight? It uses the spot pool ZopNight already cleaned up. Day and Night talk to each other.
