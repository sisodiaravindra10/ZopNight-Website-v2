// cursor variants · 18 examples

Eighteen pointers. One brand.

Native CSS cursors plus custom SVG cursors built from the ZopDev mark, dot, scan reticle, and pen-tip primitives. Hover any demo zone to see the cursor change. The "css" line in each card is the rule you'd drop into your stylesheet.

hover · default
C1 · DEFAULT
System arrow
cursor: default;
The OS arrow. Use anywhere there's no specific affordance, body copy, page chrome, ambient surfaces.
hover · pointer
C2 · POINTER
Click hand
cursor: pointer;
Clickable elements, links, buttons, cards. Anything that triggers a navigation or action on click.
hover · crosshair
C3 · CROSSHAIR
Target / deploy here
cursor: crosshair;
Geometry that needs precise placement, drop-pin maps, region selectors, deploy markers.
hover · text
C4 · TEXT
I-beam
cursor: text;
Selectable text, paragraphs, code blocks, input fields. The browser auto-applies this on `<input>` / `<textarea>`.
hover · move
C5 · MOVE
Four-arrow translate
cursor: move;
Draggable along both axes, kanban cards, movable panels, dashboard widgets being repositioned.
press · grab
C6 · GRAB
Open hand → fist
cursor: grab; & :active { cursor: grabbing }
Pannable surfaces, maps, canvases, image viewers. The press state flips to `grabbing` automatically on the same element.
hover · help
C7 · HELP
Question mark
cursor: help;
Tooltipped or info-on-hover elements. Pair with a small `(i)` glyph in the UI for redundancy.
hover · wait
C8 · WAIT
Spinner / hourglass
cursor: wait;
Synchronous OS-blocking work. Avoid, prefer in-page skeletons or progress indicators. Reserve for <2s waits.
hover · not-allowed
C9 · NOT-ALLOWED
No-entry
cursor: not-allowed;
Disabled actions, submit buttons before form valid, locked rows, gated CTAs. Always pair with `aria-disabled="true"`.
hover · zoom-in
C10 · ZOOM-IN
Magnify
cursor: zoom-in;
Click-to-magnify surfaces, chart drilldowns, image lightboxes, dashboard tiles that expand. Use `zoom-out` for the inverse.
hover · orange dot
C11 · BRAND DOT
ZopDev orange anchor
cursor: url(...orange-square...) 8 8, default;
A 6×6 brand-orange square. Cleanest possible custom cursor, the smallest mark of brand presence on hover-rich surfaces.
hover · brand mark
C12 · BRAND MARK
4-quadrant lockup
cursor: url(...mark-zopdev...) 10 10, pointer;
The full ZopDev mark (blue / orange / green) at 20px. Reserve for high-signature surfaces, hero, brand-led campaign pages.
hover · scan reticle
C13 · SCAN RETICLE
Targeting reticle
cursor: url(...reticle...) 14 14, crosshair;
Ink ring with cross-hair lines + brand-orange centre dot. Operator-style, for region selectors, monitoring drilldowns, "scan this".
hover · pen tip
C14 · PEN TIP
Annotate / edit
cursor: url(...pen...) 2 2, pointer;
Sharp ink chevron with a paper-coloured stroke for contrast. For annotation surfaces, comment threads, mark-up tools.
hover · plus
C15 · PLUS
Add / spawn
cursor: url(...plus...) 11 11, pointer;
Paper disc with ink outline + orange `+`. Reserve for "click to add" zones, empty states, blank canvases, drop-targets.
hover · minimal dot
C16 · MINIMAL DOT
Operator default
cursor: url(...mini-dot...) 5 5, default;
A tiny 4×4 orange square. Smaller and quieter than C11. Ambient surfaces where you want subtle brand presence without intrusion.
hover · ZopNight
C17 · ZOPNIGHT
Blue tile + moon
cursor: url(...zopnight-mark...) 9 9, pointer;
The ZopNight sub-brand mark as a cursor. Use only on ZopNight-specific surfaces (cost optimisation, infra views).
hover · ZopCloud
C18 · ZOPCLOUD
Pixel cloud
cursor: url(...zopcloud-mark...) 9 9, pointer;
8-bit pixel-cloud mark in the brand quadrant colours. Use on ZopCloud-specific surfaces, multi-cloud platform pages.
