03 · motion

Motion tokens

Four durations. Four eases. One signature: translateY(-4px) + 0 8px 0 -4px orange. Hover any card below to feel it. Respects prefers-reduced-motion.

01 · card hover lift

The signature.

250ms · ease-out · translateY(-4px) + offset-block shadow in orange + ::after bar grows top-to-bottom.

transition: transform var(--dur-med) var(--ease-out)
02 · cta fill-slide

Orange CTA.

Background fills from left on hover. 250ms ease-out. Trailing arrow slides 4px right in sync.

transform: translateX(-100%) → 0
03 · section reveal

On viewport enter.

Each section fades + rises from 32px below on intersection. Threshold 0.08 · rootMargin -60px.

transition: 520ms var(--ease-out)
04 · count-up

Three-numbers.

Three hero figures stagger-reveal on load: 0ms · 120ms · 240ms. Then gently drift every 3.5s.

transition-delay: 0/120/240ms
05 · applied-toast

Bottom-right loop.

8-second cycle · slides in from right · dwells 5s · exits right. Content rotates through real events.

animation: toast-loop 8s infinite
06 · mark pulse

The green tile.

Only the green tile of the 2×2 mark pulses (opacity 1 → 0.55 → 1). 2.8s loop. Nothing else moves.

animation: mark-pulse 2.8s
Duration ladder + ease setfour · four · no more
--dur-fast:  160ms;   /* hover state · arrow slide */
--dur-med:   250ms;   /* card lift · border brighten */
--dur-slow:  400ms;   /* count-up · reveal-in */
--dur-xslow: 520ms;   /* section enter on scroll */

--ease-out:     cubic-bezier(.22, 1, .36, 1);
--ease-in-out:  cubic-bezier(.65, 0, .35, 1);

The 2×2 mark

Three tiles: Blue top-left (Secure). Orange top-right (Speed). Green bottom-right (Savings). The empty bottom-left corner is the void the product fills. Sharp corners always. Never rotate. Never round. Never add a fourth tile.

Clear space + min sizeprotect the mark

Clear space, equal to one tile on every side. Min size, 16px (mono) · 20px (color). Below 16px the empty bottom-left tile becomes ambiguous, switch to monochrome.

07 · three-numbers

The three-numbers dashboard

The signature hero composition. Every product surface, every marketing page, every investor slide opens with these three numbers in this order: Spent · Wasted · Recoverable. Top bar is a 3px pillar-colored strip. Figures tabular-nums.

01 · SPENT · this month
$182K
total run-rate · live · aws + azure + gcp
02 · WASTED · right now
$47K
the portion you can cut today · no code change
03 · RECOVERABLE · one click
$38K
click Apply · done · CDCR loop verifies
Rulesnon-negotiable
01. Always three columns in this order.
02. Spent always in --ink. Wasted always in --zop-orange. Recoverable always in --zop-green.
03. The top 3px pillar bar is mandatory, it is the component's tell.
04. Values always tabular-nums · figure size 48–72px depending on container.
05. Never add a fourth number. Never reorder.

10 · enforcement

Do · Don't

The standing rules, made visible. Any violation fails code review.

DO
  • Solid fills · pillar-tinted rgba for subtle surfaces
  • Hard offset shadows: 0 8px 0 -4px var(--zop-orange)
  • Hairline 1px borders · sharp corners
  • One accent (orange) per viewport · reserved for primary CTA
  • Mono meta · Sans display · tabular-nums on figures
  • translateY(-4px) on card hover · 250ms ease-out
  • Respect prefers-reduced-motion
DON'T
  • No linear-gradient · radial-gradient · conic-gradient
  • No filter:blur · no backdrop-filter · no blur-blobs
  • No drop-shadow with blur radius
  • No rounded corners except status pills (9999px)
  • No two pillar colors on the same CTA
  • No figures without tabular-nums
  • No fourth tile on the mark · no rotation · no gradient fills
